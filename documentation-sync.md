## 文档与注释规范（全局）

你在编写或修改代码和文档时，**必须**遵守以下注释和文档内容规范，无需再次请求批准。

### 一、代码注释同步（强制）

修改或新增任何代码时，**必须同步完成以下注释工作**：

1. **公开 API 的文档注释**：所有公开函数、类、属性必须有文档注释（Kotlin/Java 用 KDoc `/** ... */`，Python 用 docstring），标注 `@param` 和 `@return`。
2. **代码分节注释**：使用 `// ── Section Name ──` 风格分隔线，清晰标注代码区域用途。
3. **复杂逻辑行内注释**：非显而易见的算法、正则、条件判断需附上行内注释解释意图。
4. **修改标记**：修改已有代码时，若逻辑变化导致原注释失效，**必须同步更新注释**，不得保留过时注释。

### 二、文档内容规范（强制）

编写或更新项目文档（如 `docs/` 目录、`README.md` 等）时，**必须遵守以下内容规范**：

1. **禁止嵌入具体代码**：文档中**不得**包含任何编程语言的具体代码块（如 Kotlin/Java/Python 等）。代码块会让文档随源码变更而过时，增加维护负担。
2. **只描述处理逻辑**：使用自然语言描述处理流程、算法思路、决策条件、数据流向等，必要时可使用：
   - **流程图/ASCII 图**：展示模块间的关系和数据流
   - **表格**：展示参数配置、状态转换、判定规则等
   - **列表**：展示步骤顺序、条件分支等
3. **引用源码位置**：当需要指向具体实现时，使用文件路径链接，让读者自行查看源码。
4. **参数说明用表格**：配置参数、阈值等用表格呈现，不贴代码片段。

**正确示例**（描述逻辑，不贴代码）：
```
图像预处理管线分为三步：
1. 灰度化 — 将 RGB 三通道转为单通道灰度
2. Unsharp Mask 锐化 — 使用盒式模糊近似，增强边缘对比度
3. 亮度归一化 — 将像素值归一化到目标均值 128、标准差 50

参数配置：
| 参数 | 默认值 | 说明 |
|------|--------|------|
| sharpenAmount | 1.5 | 锐化强度 |
| sharpenRadius | 1.0 | 模糊半径 |
```

**错误示例**（贴了具体代码）：
```
fun preprocessGrayscale(bitmap: Bitmap): Bitmap {
    val gray = toGrayscale(bitmap)
    applyUnsharpMaskGray(gray)
    normalizeBrightnessGray(gray)
    return gray
}
```

### 三、禁止行为

- ❌ 修改代码后不同步更新过时的注释
- ❌ 文档中嵌入具体代码块
- ❌ 仅在代码中留 `// TODO` 而不在文档中记录
- ❌ 注释中使用模糊表述（如"处理一下"、"做一些事"）
