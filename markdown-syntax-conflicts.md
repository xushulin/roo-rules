# Markdown 文档标点符号与语法冲突规避规则（全局）

你在编写或修改任何 Markdown 文件时，**必须**遵守以下规则，防止文档中的文字内容标点被 Markdown 解析器误判为语法标记，无需再次请求批准。

## 一、核心原则

Markdown 保留了一组字符作为**语法标记**：`|`、`~`、`*`、`_`、`<`、`>`、`` ` ``、`#` 等。当文档的**文字内容**中需要出现这些字符（且它们不是 Markdown 语法的一部分）时，必须用替代写法避免解析冲突。

## 二、表格内的冲突（强制）

### 2.1 竖线 `|` — 与表格列分隔符冲突

**场景**：Markdown 表格以 `|` 作为列分隔符。单元格内出现裸 `|` 会导致表格结构错乱。

**替代方案**（按优先级排序）：

| 方案 | 写法 | 适用场景 |
|------|------|---------|
| **文字描述**（首选） | `半角竖线`、`Shift + \` | 分隔符类说明（如"各组之间用半角竖线连接"） |
| HTML 实体 | `&#124;` | 必须在单元格内显示 `|` 字符本身时 |
| 行内代码 | `` `\|` `` | 示例值中的分隔符，无歧义时可使用 |

**注意**：`&#124;` 在某些 Markdown 预览器中会显示为字面字符串而非 `|` 符号，因此仅在必须逐字显示 `|` 符号时才使用；普通说明优先用文字描述。

### 2.2 波浪号 `~` — 与下标语法冲突

**场景**：部分 Markdown 渲染器（如 Pandoc、GitHub Flavored Markdown 的部分扩展）将 `~text~` 解析为下标标记。

**替代方案**：

| 场景 | 写法 | 示例 |
|------|------|------|
| 数值区间（首选） | 全角 `～`（U+FF5E） | `30～50`、`5～10` |
| 行内代码内的区间 | 半角 `~` 可保留 | `` `30~50` `` |

### 2.3 星号 `*` — 与强调/列表语法冲突

**场景**：`*text*` → 斜体，`**text**` → 粗体。单元格内含 `*` 如 `必须 > 0` 中的 `**` 易触发加粗解析。

**替代方案**：

| 场景 | 写法 |
|------|------|
| 强调标记 `**` 出现在单元格文本中 | 用 `\*\*` 转义；或改用全角 `＊`（仅在纯中文语境可用） |
| `*` 列表中作为项目符号 | 不在表格单元格内使用 `* ` 开头的行 |

### 2.4 尖括号 `<` `>` — 与 HTML 标签冲突

**场景**：`<text>` 被解析为 HTML 标签，且通常不会显示。

**替代方案**：

| 场景 | 写法 |
|------|------|
| 必须显示字面 `<` `>` | HTML 实体 `<` `>` |
| 示例/代码值 | 用行内代码包裹如 `` `<input>` `` |
| 中文书名号 | 优先使用全角书名号 `《》` 替代尖括号 |

## 三、非表格区域的冲突（建议）

以下情况在表格外也应注意，虽然不一定破坏结构，但可能导致格式异常：

| 字符 | 风险 | 建议 |
|------|------|------|
| `~` | 下标标记（Pandoc） | 区间用全角 `～` |
| `*` / `_` 包围的文本 | 意外斜体/粗体 | 用 `\` 转义或加空格 |
| `<` `>` 包围的文本 | 被当作 HTML 标签 | 用 `<` `>` |
| `#` 在行首 | 被当作标题 | 缩进或用 `\#` |

## 四、行内代码的豁免

被反引号 `` ` `` 包裹的**行内代码**内的任何字符都**不受** Markdown 语法解析影响。因此：

- ✅ `` `30~50Hz` `` — `~` 在行内代码内，安全
- ✅ `` `\` `` 键上方 `` — `\` 在行内代码内，安全
- ✅ `` `1,2|2,3` `` — `|` 在行内代码内，安全

行内代码是表达**具体输入值格式**的最佳载体，应优先使用。

## 五、自检清单

每次修改 `.md` 文件后，自问：

- [ ] 表格单元格中是否有裸 `|`？如有，是否已替换为文字描述或 `&#124;`？
- [ ] 表格单元格中是否有裸 `~` 表示数值区间？如有，是否已替换为全角 `～`？
- [ ] 表格单元格中是否有 `**` 或 `*文本*` 可能触发意外强调？是否已转义？
- [ ] 是否有 `<` `>` 包围的非 HTML 文本？是否已转为实体或行内代码？
- [ ] 行首是否有裸 `#` 被误当作标题？
- [ ] 符号类的具体输入值是否已用行内代码包裹？

## 六、禁止行为

- ❌ 在表格单元格内直接写裸 `|` 作为文字内容的分隔符说明
- ❌ 用半角 `~` 表示区间（如 `5~10`），尤其在表格中
- ❌ 在表格单元格内使用未转义的 `<` `>` 作为文字内容
- ❌ 用 `*` 或 `_` 包裹非强调性的文字内容
- ❌ 知道冲突存在但仍然不改，等渲染出问题再修
